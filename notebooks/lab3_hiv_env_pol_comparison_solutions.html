All these sequences are aligned, which means that they are the same length. The '-' in the sequences have been placed in the most appropriate locations so that sequences line up. These dashes represent base pairs that were potentially inserted or deleted, 'indels'.

The procedure of putting in these dashes in an automated and rigorous manner is called alignment. Although there are standard algorithms for alignment, there is no guarantee that the alignment produced represents the true mutational events that occurred.

However, because HIV-1 is so well characterized, we can more or less trust these alignments.

Quantifying variation II¶

One of the problems of only counting the number of variants is that rare mutants get as much weight as common ones. We can try to get around this by looking at the relative frequencies of each variant and reporting the frequency of the most common variant for each position.

Let's practice this on the first position of the env gene.

In [31]:
first_pos=np.array([l[0].upper() for l in env_seqs])

Q7: Write code that returns the frequency of the most common variant in first_pos.

One way to do this is to first make a list of the unique variants. Then, for each unique variant, count the number of times it appears in first_pos. Finally, divide the count by the length first_pos to get a fraction.

In [ ]:

Q8: Now put this code in a loop that scans the full env gene and stores the max frequencies for each position in the array env_max_freq.

In [33]:
env_max_freq=np.zeros((env_len,))
for pos in range(env_len):
    bases=np.array([l[pos].upper() for l in env_seqs])
    variants=set(bases)
    if len(variants)==1:
        env_max_freq[pos]=1 # all are the same type
    else:
        freqs=[]
        for v in variants:
            freqs.append(np.sum((bases==v))/len(bases))
        env_max_freq[pos]=max(freqs)
