Notice that for each locus, there are two columns. For example, 'AMEL' and 'AMEL.1'. This is because there is a reading from each copy of the chromosome. In the special case for 'AMEL', the type, X or Y, tells us whether the individual is XX or XY. The columns for the other loci contain measurements of the number of copies (allele) of each STR.

Restricting to CODIS core loci¶

The FBI use the following 15 STR loci for DNA fingerprinting: TPOX, D3S1358, FGA, D5S818, CSF1PO, D7S820, D8S1179, TH01, VWA, D13S317, D16S539, D18S51, D21S11, AMEL.

Only the last STR locus, AMEL, is on the sex chromosomes. Create a new table, call it CODIS_profiles, which only contains the DNA type for the CODIS core loci for each individual. Keep the 'Group' column also.

To keep things simple, we will ignore the columns with name of the form XXXXXXX.1 and we'll also ignore the first two columns which contains ethnic information and the unique identifier.

Upper bound on the match probability¶

Assuming that the distribution of alleles for each STR locus in this sample is representative of the population, we can use it to estimate the probability that a suspect's DNA matches the DNA at the crime scene just by chance. For this exercise, we will not be using the 'AMEL' column.

If we drew individuals at random from the population, it would be reasonable to expect that their alleles for the 'TPOX' STR locus would be distributed approximately as:

Optional: Degree of similarity between individuals¶

For these last two questions, you may want to consider converting CODIS_profiles into a numpy array (code below). Also, I've provided a function that outputs the distance between two fingerprints (it's just Euclidean distance).

In [12]:
def euclideanDist(fingerprint1, fingerprint2):
    '''given two DNA fingerprints (vectors of the same length) outputs the Euclidean distance'''
    return(np.sum((np.array(fingerprint1)-np.array(fingerprint2))**2)**.5)
