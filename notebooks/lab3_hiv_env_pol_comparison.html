All these sequences are aligned, which means that they are the same length. The '-' in the sequences have been placed in the most appropriate locations so that sequences line up. These dashes represent base pairs that were potentially inserted or deleted, 'indels'.

The procedure of putting in these dashes in an automated and rigorous manner is called alignment. Although there are standard algorithms for alignment, there is no guarantee that the alignment produced represents the true mutational events that occurred.

However, because HIV-1 is so well characterized, we can more or less trust these alignments.

Quantifying variation I¶

There are many ways to quantify how much variation we see in a sample. One way is to look at each position and see how many variants there are.

Let's do this for the first four positions.

Quantifying variation II¶

One of the problems of only counting the number of variants is that rare mutants get as much weight as common ones. We can try to get around this by looking at the relative frequencies of each variant and reporting the frequency of the most common variant for each position. This takes a little more code.

In [13]:
env_len=3576
env_max_freq=np.zeros((env_len,))
for pos in range(env_len):
    bases=np.array([l[pos].upper() for l in env_seqs])
    variants=set(bases)
    if len(variants)==1:
        env_max_freq[pos]=1 # all are the same type
    else:
        freqs=[]
        for v in variants:
            freqs.append(np.sum((bases==v))/len(bases))
        env_max_freq[pos]=max(freqs)

def hammingDistance(str1, str2):
    '''str1 and str2 are two character strings of the same length
    returns the number of positions str1 and str2 differ (integer)'''
    
    if len(str1)!=len(str2):
        print('Please enter two strings of the same length.')
        return
    else:
        return(sum([(c1!=c2) for c1, c2 in zip(str1, str2)]))
